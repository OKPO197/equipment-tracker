# Учёт оргтехники — веб-версия

Этот проект — та же система учёта техники и расходников, что была в артефакте
Claude, но переведённая на реальный веб-хостинг (GitHub Pages) с базой данных
Firebase вместо внутреннего хранилища Claude.

Ниже — пошаговая инструкция. Все шаги нужно выполнить один раз при первой
настройке.

---

## Часть 1. Создаём базу данных в Firebase (бесплатно)

1. Зайдите на **https://console.firebase.google.com** и войдите с Google-аккаунтом.
2. Нажмите **«Add project» / «Создать проект»**.
   - Введите любое название, например `equipment-tracker`.
   - На вопрос про Google Analytics можно ответить «Отключить» — не нужна.
3. Когда проект создался, в левом меню откройте **Build → Firestore Database**.
   - Нажмите **«Create database»**.
   - Выберите ближайший регион (например, `eur3` для Европы).
   - На вопрос про режим безопасности выберите **«Start in production mode»**
     (правила безопасности мы зададим сами на шаге 5).
4. В левом меню откройте **Build → Authentication**.
   - Нажмите **«Get started»**.
   - На вкладке **Sign-in method** включите провайдер **«Anonymous»**
     (это единственный способ входа, который нужен — сам он ничего не
     спрашивает у пользователя, просто даёт браузеру технический идентификатор).
5. Всё ещё в Firebase: слева найдите **Firestore Database → Rules** (вкладка
   сверху) и вставьте туда содержимое файла **`firestore.rules`** из этого
   проекта, затем нажмите **Publish**.
6. Получите конфигурацию веб-приложения:
   - В левом меню нажмите на **шестерёнку → Project settings**.
   - Прокрутите вниз до **«Your apps»**, нажмите значок **`</>`** (Web).
   - Введите любое имя (например, `web`), «Also set up Firebase Hosting» —
     **не отмечайте** (хостинг у нас будет через GitHub Pages).
   - Firebase покажет блок кода с объектом `firebaseConfig = { apiKey: ..., ... }`.
     Скопируйте эти значения.
7. Откройте файл **`src/firebaseConfig.js`** в этом проекте и вставьте туда
   скопированные значения вместо `"YOUR_API_KEY"` и так далее.

Firebase настроен.

---

## Часть 2. Кладём проект на GitHub

1. Зайдите на **https://github.com/new** и создайте новый репозиторий
   (например, `equipment-tracker`). Можно приватный — GitHub Pages работает
   и с приватными репозиториями на платных тарифах, либо сделайте публичным,
   если это не проблема (сам сайт всё равно защищён логином внутри
   приложения — см. предупреждение в конце).
2. Откройте файл **`vite.config.js`** и замените `REPO_NAME` на точное имя
   вашего репозитория, например `/equipment-tracker/`.
3. Загрузите все файлы этого проекта в свежесозданный репозиторий — самый
   простой способ, если вы не работали с git раньше:
   - Установите **GitHub Desktop** (https://desktop.github.com) — это
     приложение с кнопками, без командной строки.
   - Откройте GitHub Desktop → «Add local repository» → укажите папку
     с этим проектом на вашем компьютере.
   - Нажмите «Publish repository» и выберите созданный на шаге 1 репозиторий.

   Если вы работаете через терминал:
   ```
   git init
   git add .
   git commit -m "Первая версия"
   git branch -M main
   git remote add origin https://github.com/ВАШ_АККАУНТ/ВАШ_РЕПОЗИТОРИЙ.git
   git push -u origin main
   ```

4. На GitHub откройте вкладку **Settings → Pages** в вашем репозитории.
   - В разделе **«Build and deployment»** → **Source** выберите
     **«GitHub Actions»** (не «Deploy from a branch»).
5. Зайдите во вкладку **Actions** репозитория — там должен запуститься (или
   уже быть готов) workflow **«Deploy to GitHub Pages»**. Он сам собирает
   проект и публикует его при каждом изменении в ветке `main`.
6. Через 1–2 минуты после первого успешного запуска на вкладке
   **Settings → Pages** появится готовая ссылка вида:
   ```
   https://ВАШ_АККАУНТ.github.io/ВАШ_РЕПОЗИТОРИЙ/
   ```
   Это и есть рабочий адрес сайта — открывается в любом браузере, на любом
   устройстве, без какого-либо отношения к Claude.

---

## Как дальше вносить изменения

Если что-то нужно поправить в самом приложении — просто отредактируйте
`src/App.jsx` и отправьте изменения на GitHub (`git push` или через GitHub
Desktop). Сайт пересоберётся и обновится автоматически в течение минуты-двух.

## О безопасности (важно прочитать)

- Внутри приложения по-прежнему работает своя система логинов/паролей
  (вкладка «Пользователи»), которую вы настраивали раньше — это не изменилось.
- Ключ `firebaseConfig` (apiKey и т.п.) — это не секрет, его *можно* видеть
  всем, кто откроет сайт: доступ к данным регулируется не им, а правилами
  Firestore (`firestore.rules`), которые разрешают чтение/запись только
  клиентам, прошедшим (анонимный) вход.
- Это значит: **любой, у кого есть ссылка на сайт**, технически может открыть
  инструменты разработчика в браузере и напрямую читать/менять данные в
  Firestore, минуя окно входа в приложение — так же, как это было возможно
  и в версии на Claude. Это лёгкая защита для внутреннего инструмента отдела,
  а не защита от целенаправленной атаки. Если захотите более строгий уровень
  защиты (например, чтобы доступ к самой базе был только у авторизованных по
  логину/паролю сотрудников, а не у любого анонимного клиента) — это можно
  сделать, но потребует более глубокой переделки на настоящую Firebase
  Authentication с именными аккаунтами. Скажите, если это нужно.
